# 单身期家庭资产诊断 MVP 设计

## 目标

本文档定义生命周期分层家庭资产诊断模型的第一版 MVP。长期目标是根据家庭财务报表识别资产结构、现金流、负债、保障和成长问题，并把诊断结果转化为资产配置与改善建议。

第一版先聚焦“单身期”，使用两张家庭财务报表作为输入：

- 家庭资产负债表
- 家庭收支储蓄表

模型不能只输出分数，还应识别风险、解释原因，并给出有优先级的改善动作。

## 设计原则

- 生命周期分层：单身期有独立的权重、阈值和风险优先级。后续生命周期复用同一架构，但覆盖参数。
- 底层诚实，展示友好：计算层严格透明；前端可以把分数表达得更温和，但风险解释和建议必须如实。
- 先安全，再增长：单身期优先关注流动性、储蓄能力和偿债压力，再关注成长能力和财富自由度。
- 避免重复计分：数学上高度互补的指标不同时参与同一维度评分。
- 参数化优先：指标定义、权重、阈值、S 曲线参数和诊断规则都应配置化，避免写死。
- MD 与 Excel 同步交付：每次形成或更新模型规格时，应尽量同步生成 Excel 工具版本，方便用样本家庭测试参数和诊断话术。
- 体现传导逻辑：诊断不能只看孤立维度，还应解释维度之间的影响链条，例如储蓄能力弱会限制投资能力，偿债压力高会挤压流动性和成长空间。

## MVP 输出

模型第一版应输出：

1. 家庭资产健康总分
2. 六个维度分数
3. 关键风险提示
4. 低分维度和低分指标的原因解释
5. 有优先级的改善动作
6. 高层次资产结构调整方向
7. 维度之间的连锁诊断，例如“储蓄能力不足 -> 成长能力受限”或“偿债压力偏高 -> 流动性风险上升”

MVP 暂不输出具体金融产品、保险产品或投资标的建议。

## 模型架构

### 1. 数据输入模块

数据输入模块从两张家庭财务报表中采集并标准化字段。

基础输入包括：

- 总资产
- 总负债
- 净资产
- 流动资产
- 短期负债
- 生息资产
- 可投资资产
- 长期投资资产
- 年收入
- 年支出
- 月必要支出
- 固定支出
- 年债务本息支出
- 高息消费贷余额
- 年保费
- 基础保障配置项
- 有效重大风险保障额度
- 年被动收入
- 年投资收益
- 期初投资资产

生命周期和场景元数据包括：

- 生命周期阶段：MVP 固定为单身期
- 收入稳定性：稳定、一般、不稳定
- 城市成本压力：低、中、高
- 重大目标：无、买房、结婚、进修、创业、其他

v0.1 中场景元数据可以先不强制参与评分，但模型结构要为后续场景修正预留接口。

### 2. 指标计算模块

指标计算模块先计算原始比率，再通过参数化 S 曲线转化为指标得分。

默认 S 曲线：

```text
Score = 100 / (1 + EXP(-k * (x - m)))
```

其中：

- `x` 是原始指标值
- `m` 是 50 分中点
- `k` 控制曲线陡峭程度
- `k` 为正表示越高越好
- `k` 为负表示越低越好

对于“区间最优”指标，例如保费收入比，应使用分段函数，而不是单一 S 曲线。

### 3. 诊断规则模块

诊断规则模块负责解释分数和硬性风险。

每条诊断应包含：

- 风险等级：绿灯、黄灯、红灯
- 通俗解释
- 用户数据证据
- 关联维度
- 是否覆盖普通分数解释

硬性风险规则必须独立于加权评分进行计算，避免严重风险被平均分稀释。

### 4. 传导逻辑模块

传导逻辑模块负责识别维度之间的影响链条。它不直接替代六维评分，而是在报告中解释“为什么这个问题会进一步影响其他能力”。

MVP 先支持以下传导关系：

| 起点维度 | 传导到 | 触发条件 | 诊断含义 |
| --- | --- | --- | --- |
| 储蓄能力 | 成长能力 | 储蓄能力低于 60 分，或自由储蓄率低于 10% | 新增可投资资金不足，长期资产配置改善速度受限 |
| 储蓄能力 | 流动性 | 总结余率低于 10%，且紧急备用金低于 3 个月 | 很难靠未来现金流快速补足安全垫 |
| 偿债能力 | 流动性 | 债务偿付率高于 30%，或高息负债收入比高于 10% | 还款占用现金流，收入波动时更容易出现短期资金缺口 |
| 偿债能力 | 成长能力 | 债务偿付率高于 30%，且长期投资资产占比低于 50% | 债务支出挤压长期投资，资产增长速度受限 |
| 保障能力 | 流动性 | 基础保障完整度低于 60%，且紧急备用金低于 3 个月 | 一旦发生医疗、意外或收入中断事件，备用金可能被快速消耗 |
| 流动性 | 成长能力 | 紧急备用金低于 3 个月，且投资资产占比较高 | 安全垫不足时继续投资，可能导致市场下跌或突发支出时被迫卖出资产 |
| 成长能力 | 财富自由度 | 成长能力低于 60 分，且被动收入覆盖率低于 10% | 资产缺乏增值和现金流贡献，财富自由进度偏慢 |

传导诊断应包含三层信息：

1. 起点问题：哪个维度或指标首先出现短板
2. 影响路径：它会如何影响其他维度
3. 优先行动：应先修复哪一个前置条件

示例：

```text
强传导：储蓄能力 -> 成长能力
原因：自由储蓄率偏低，新增可投资资金不足。
影响：即使当前投资配置合理，长期资产规模也难以持续扩大。
优先行动：先提高月度结余，再逐步增加长期投资资产占比。
```

### 5. 建议映射模块

建议映射模块把诊断结果转成行动优先级。

MVP 的建议应是方向性建议，而不是产品级建议：

- 建立紧急备用金
- 降低高息负债
- 改善现金流结余
- 补齐基础保障
- 在流动性安全后，把闲置资金逐步转入长期资产
- 降低支出刚性
- 避免杠杆投资

后续版本可以把这一模块扩展为资产配置建议引擎。

### 6. Excel 工具模块

MVP 应同步生成 Excel 文件，作为参数测试和业务验证工具。Excel 不只是静态表格，应尽量包含可计算公式和示例家庭。

建议工作表：

| 工作表 | 用途 |
| --- | --- |
| README | 说明模型用途、表结构和使用方式 |
| 示例输入 | 提供一个中产单身家庭样本，用于验证参数和诊断逻辑 |
| 指标参数 | 存放六维指标、权重、公式口径、方向、m、k 和锚点 |
| 评分计算 | 根据示例输入计算原始指标值、指标得分、维度得分和红黄绿状态 |
| 六维总览 | 展示综合分、六维分数、状态和一句话诊断 |
| 连锁诊断 | 展示维度之间的传导关系、触发条件、影响解释和优先行动 |
| 建议库 | 存放可复用的建议话术和行动优先级 |

后续每次修改 MD 中的指标、权重或规则时，应同步更新 Excel。

## 单身期维度权重

| 维度 | 权重 | 定位 |
| --- | ---: | --- |
| 储蓄能力 | 22% | 收入转化为资产的核心发动机 |
| 流动性 | 20% | 防止突发事件击穿财务安全 |
| 偿债能力 | 20% | 控制负债规模和还款压力 |
| 保障能力 | 15% | 转移重大风险，避免一次事故归零 |
| 成长能力 | 15% | 判断资产是否具备长期增值能力 |
| 财富自由度 | 8% | 结果性指标，单身期低权重保留 |

综合分公式：

```text
综合分 = SUM(维度得分 * 维度权重)
维度得分 = SUM(指标得分 * 维度内权重)
```

## MVP 指标总表

百分比在表格或代码中建议用小数存储，例如 20% 存为 `0.2`。

| 维度 | 指标 | 维度内权重 | 公式 | 方向 | m | k | 经验锚点 |
| --- | --- | ---: | --- | --- | ---: | ---: | --- |
| 偿债能力 | 资产负债率 | 30% | 总负债 / 总资产 | 越低越好 | 40% | -10 | 20%优，40%中点，60%警示 |
| 偿债能力 | 债务偿付率 | 45% | 年债务本息 / 年收入 | 越低越好 | 25% | -12 | 10%优，25%中点，40%警示 |
| 偿债能力 | 高息负债收入比 | 25% | 高息消费贷余额 / 年收入 | 越低越好 | 10% | -25 | 0%优，10%警示，20%红灯 |
| 流动性 | 紧急备用金倍数 | 50% | 流动资产 / 月必要支出 | 越高越好 | 3个月 | 1.1 | 1个月红灯，3个月中点，5-6个月优 |
| 流动性 | 净流动覆盖倍数 | 25% | (流动资产 - 短期负债) / 月必要支出 | 越高越好 | 2个月 | 1.2 | 0红灯，2中点，4优 |
| 流动性 | 流动资产占比 | 25% | 流动资产 / 总资产 | 越高越好 | 15% | 12 | 5%红灯，15%中点，30%优 |
| 储蓄能力 | 总结余率 | 45% | (收入 - 支出) / 收入 | 越高越好 | 20% | 12 | 5%红灯，20%中点，35%优 |
| 储蓄能力 | 自由储蓄率 | 35% | (收入 - 固定支出 - 债务本息) / 收入 | 越高越好 | 15% | 12 | 0%红灯，15%中点，30%优 |
| 储蓄能力 | 支出刚性比率 | 20% | 固定支出 / 总支出 | 越低越好 | 60% | -8 | 40%优，60%中点，80%警示 |
| 成长能力 | 生息资产占比 | 35% | 生息资产 / 总资产 | 越高越好 | 35% | 8 | 15%红灯，35%中点，55%优 |
| 成长能力 | 长期投资资产占比 | 35% | 长期投资资产 / 可投资资产 | 越高越好 | 50% | 8 | 20%红灯，50%中点，70%优 |
| 成长能力 | 投资报酬率 | 30% | 年投资收益 / 期初投资资产 | 越高越好 | 4% | 12 | 0%低，4%中点，8%优 |
| 保障能力 | 基础保障完整度 | 35% | 已配置基础保障项 / 应配置基础保障项 | 越高越好 | 60% | 8 | 0-30%红灯，60%中点，100%优 |
| 保障能力 | 保费收入比 | 30% | 年保费 / 年收入 | 区间最优 | 5%-12% | 分段 | 低于3%可能不足，5%-12%合理，高于20%过重 |
| 保障能力 | 重大风险覆盖倍数 | 35% | 有效保障额度 / 年必要支出 | 越高越好 | 3倍 | 0.8 | 1倍红灯，3倍中点，6倍优 |
| 财富自由度 | 被动收入覆盖率 | 50% | 年被动收入 / 年支出 | 越高越好 | 10% | 8 | 0%低，10%中点，30%优 |
| 财富自由度 | 净资产支出倍数 | 50% | 净资产 / 年支出 | 越高越好 | 1倍 | 1.1 | 0低，1倍中点，3倍优 |

## 保费收入比分段评分

保费收入比是“区间最优”指标。过低可能说明风险转移不足，过高则可能挤占储蓄和投资空间。

v0.1 建议规则：

```text
若 x < 5%：
  Score = 100 / (1 + EXP(-50 * (x - 0.05)))

若 5% <= x <= 12%：
  Score = 95

若 12% < x <= 20%：
  Score = 95 - (x - 12%) / 8% * 35

若 x > 20%：
  Score = MAX(20, 60 - (x - 20%) / 10% * 40)
```

这是一组经验参数，后续需要通过样本家庭回测校准。

## 硬性风险规则

硬性风险必须单独触发，即使综合分看起来不低，也要在报告中提示。

| 风险 | 触发条件 | 处理 |
| --- | --- | --- |
| 现金流为负 | 月结余 < 0 | 储蓄能力红灯 |
| 流动性击穿 | 紧急备用金 < 1个月必要支出 | 流动性红灯 |
| 高息债风险 | 高息负债收入比 > 10% | 偿债能力警示 |
| 债务压力过高 | 债务偿付率 > 40% | 偿债能力红灯 |
| 基础保障缺失 | 无基础医保、意外或重疾类保障 | 保障能力红灯 |
| 杠杆投资 | 存在借贷投资或高杠杆交易 | 成长能力风险提示 |

## 分数展示

计算层使用严格分数。展示层可以使用更友好的标签：

| 分数区间 | 内部含义 | 前端展示建议 |
| --- | --- | --- |
| 85-100 | 强 | 健康 |
| 70-84 | 基本健康 | 基础良好 |
| 55-69 | 有短板 | 需要针对性改善 |
| 40-54 | 较弱 | 财务基础需要修复 |
| 0-39 | 高风险 | 需要优先干预 |

前端不得用友好的综合分隐藏红灯级硬性风险。

## 建议优先级逻辑

MVP 应按紧急程度排序建议：

1. 先处理急性风险：现金流为负、流动性击穿、高息负债、杠杆投资
2. 再建立安全底座：紧急备用金、基础保障、降低支出刚性
3. 提高积累效率：提升总结余率和自由储蓄率
4. 提升资产成长：在流动性和债务安全后，提高适合的长期资产占比
5. 逐步提高财富自由度：提升被动收入覆盖率和净资产支出倍数

当连锁诊断触发时，建议排序应优先修复传导链条的起点。例如，储蓄能力不足导致成长能力受限时，应先改善现金流结余，而不是直接要求用户提高投资资产占比。

示例输出：

```text
主要短板：流动性
数据证据：紧急备用金只能覆盖 0.8 个月必要支出。
风险解释：一旦出现短期收入中断，可能被迫借债或卖出投资资产。
优先行动：在增加高波动投资前，先把紧急备用金补到至少 3 个月必要支出。
```

## 后续生命周期扩展

后续生命周期复用同一架构，但覆盖以下内容：

- 维度权重
- 指标是否纳入
- 指标维度内权重
- S 曲线 `m` 和 `k`
- 硬性风险规则
- 建议映射

预计扩展阶段：

- 家庭形成期
- 子女教育期
- 家庭成熟期
- 退休前期
- 退休期

例如，子女教育期可能提高保障能力和流动性权重；退休期可能降低成长能力权重，提高现金流稳定性和医疗保障权重。

## 校准计划

v0.1 阈值是经验值，不能直接视为最终标准。后续应使用样本家庭进行回测。

优先校准样本：

- 收入稳定、无负债的年轻单身工薪族
- 有房贷压力的单身人群
- 有高息消费贷的单身人群
- 高收入但低储蓄的单身人群
- 自由职业或收入不稳定的单身人群
- 准备买房的单身人群

## 验收标准

该设计进入实施计划前，需要满足：

- 六个维度和维度权重被接受为单身期 MVP 基线
- 十七个指标被接受为第一版指标池
- 硬性风险规则独立于加权评分
- S 曲线参数作为可配置经验值处理
- MVP 建议仅输出方向性改善动作，不做产品级推荐
- Excel 工具与 MD 规格同步生成，且至少包含示例输入、指标参数、评分计算、六维总览、连锁诊断和建议库
- 连锁诊断能够解释关键维度之间的影响关系，并给出前置修复动作
